# SWEA 1216 회문2 — 모범 풀이가 더 빠른 이유

## 1. 문제 요약

- `100 x 100` 글자판(`'A','B','C'`)에서 **가로·세로 직선** 방향의 회문 중 가장 긴 길이를 구한다.
- 테스트 케이스 10개. 길이 1짜리(`A`)도 회문.

핵심은 "가장 긴 회문 **길이**"만 필요하다는 것. 회문 문자열 자체를 저장하거나 셀 필요가 없다.

---

## 2. 두 풀이의 방식 차이

### 2.1 내 풀이 — "시작점 고정 + 부분문자열 누적 검사"

[main.cpp](../main.cpp)의 구조:

- 모든 칸 `(r, c)`를 **시작점**으로 잡는다.
- 거기서 아래로 / 오른쪽으로 한 칸씩 늘려가며 `vector<char> temp`에 문자를 `push_back` 한다.
- 늘릴 때마다 `IsPelin(temp)`로 **전체 문자열을 처음부터 끝까지** 회문인지 다시 검사한다.

```cpp
temp.push_back(MAP[r][c]);
for (int l = 1; l < MXN - r; ++l) {
    temp.push_back(MAP[nr][c]);
    if (mx < temp.size() && IsPelin(temp))   // 매번 전체 재검사
        mx = temp.size();
}
```

이 방식에는 두 가지 구조적 약점이 있다.

1. **재검사 비용** — `IsPelin`은 양 끝에서 가운데로 좁히며 길이 `L`짜리를 매번 `O(L)`로 검사한다. 한 시작점에서 길이를 1→N까지 늘리면 `O(1)+O(2)+…+O(N) = O(N²)`.
2. **조기 종료 불가** — 시작점을 고정하면 회문 여부가 **단조롭지 않다**. 예: `AB`(아님) → `ABA`(회문) → `ABAB`(아님) → `ABABA`(회문). 한 번 깨졌다고 멈출 수 없어, 항상 끝까지 다 늘려봐야 한다.

추가로 칸마다 `vector<char>`를 할당·`push_back`(재할당 가능)·`clear` 하므로 **상수 배수(동적 할당) 부담**도 크다.

복잡도: 칸 `N²` × (방향당 `O(N²)`) = **`O(N⁴)`**.
`N=100`이면 한 TC당 약 `10⁸`, 10개 TC면 `10⁹`에 동적할당 오버헤드까지 → TLE 위험.

### 2.2 모범 풀이 — "중심 확장(expand around center)"

각 칸을 **회문의 중심**으로 보고 바깥쪽으로 대칭인지 한 칸씩 확장한다.

```cpp
// 홀수 길이: 칸 x를 중심으로
for (int i = 1; i < 50; i++) {
    if (x-i >= 0 && x+i < 100 && a[y][x-i] == a[y][x+i])
        size += 2;
    else
        break;            // ← 깨지는 순간 멈춤
}
// 짝수 길이: x-1과 x 사이를 중심으로 (size2)
```

차이를 만드는 두 지점:

1. **증분 검사 `O(1)`** — 안쪽이 이미 회문임이 보장된 상태에서 **새로 추가되는 양쪽 한 쌍만** 비교한다(`a[y][x-i] == a[y][x+i]`). 전체를 다시 훑지 않는다.
2. **조기 종료 가능** — 중심에서 확장하면 회문 여부가 **단조롭다**. 한 번 `a[x-i] != a[x+i]`로 깨지면 더 벌려도 절대 회문이 안 되므로 `break`. 그래서 한 중심당 비용이 최악 `O(N)`.

또한 `int` 카운터와 배열 인덱스 비교만 쓰므로 **동적 할당이 전혀 없다**.

복잡도: 칸 `N²` × (중심당 `O(N)`) = **`O(N³)`**.
`N=100`이면 한 TC당 약 `10⁶`, 10개 TC면 `10⁷` → 여유롭게 통과.

## 3. 왜 더 빠른가 — 정리

| 구분 | 내 풀이 (시작점 누적) | 모범 풀이 (중심 확장) |
|---|---|---|
| 한 칸당 검사 방식 | 길이마다 **전체 재검사** `O(L)` | 새 한 쌍만 **증분 비교** `O(1)` |
| 회문 단조성 | 비단조 → **조기 종료 불가** | 단조 → **깨지면 `break`** |
| 자료구조 | 칸마다 `vector` 할당/`push_back` | `int` 카운터·배열 인덱스만 |
| 시간 복잡도 | **`O(N⁴)`** (~`10⁸`/TC) | **`O(N³)`** (~`10⁶`/TC) |

세 줄 요약:

1. **알고리즘 차원이 한 단계 낮다.** 중심 확장은 "이미 회문인 안쪽"을 재활용해 검사를 `O(1)`로 줄여 `N` 한 겹을 통째로 없앤다(`N⁴ → N³`).
2. **조기 종료가 된다.** 중심 기준이면 회문성이 단조라 첫 불일치에서 멈출 수 있다. 시작점 기준은 회문성이 들쭉날쭉해 끝까지 확인해야 한다.
3. **상수 배수가 작다.** 동적 할당·`push_back` 없이 정수와 배열 비교만 쓴다.

## 4. 내 풀이를 같은 급으로 끌어올리려면

지금 구조를 유지하되 **`vector`/`IsPelin` 재검사 대신 중심 확장**으로 바꾸면 된다. 길이만 필요하므로 문자열을 모을 이유가 없다.

```cpp
// (y, x)를 중심으로 dir 방향(0:가로, 1:세로) 최장 회문 길이
int Expand(char (*map)[MXN + 1], int y, int x, int dir)
{
    int dy = MV_Y[dir], dx = MV_X[dir];
    int best = 1;                              // 홀수 중심(자기 자신)
    for (int i = 1; ; ++i) {
        int ly = y - dy * i, lx = x - dx * i;
        int ry = y + dy * i, rx = x + dx * i;
        if (ly < 0 || lx < 0 || ry >= MXN || rx >= MXN) break;
        if (map[ly][lx] != map[ry][rx]) break;
        best += 2;
    }
    int even = 0;                              // 짝수 중심(y,x 와 다음 칸 사이)
    for (int i = 0; ; ++i) {
        int ly = y - dy * i,       lx = x - dx * i;
        int ry = y + dy * (i + 1), rx = x + dx * (i + 1);
        if (ly < 0 || lx < 0 || ry >= MXN || rx >= MXN) break;
        if (map[ly][lx] != map[ry][rx]) break;
        even += 2;
    }
    return max(best, even);
}
```

이미 선언해 둔 `MV_Y/MV_X`로 가로·세로를 한 함수로 처리할 수 있어 코드도 짧아진다.

> 참고: 더 욕심내면 **Manacher** 알고리즘으로 한 줄(길이 N)을 `O(N)`에 처리해 전체 `O(N²)`까지 가능하지만, 이 문제는 `N=100`이라 `O(N³)` 중심 확장으로 충분하다.
