# 오토마타(Automata)로 문제 풀기 — 상태기계 0지식 학생용 보고서

> 이 문서는 "오토마타"라는 단어를 처음 듣는 학생도 따라올 수 있게 쓴 보고서입니다.
> 배열, 반복문, 그리고 DP·BFS를 한 번이라도 써봤다면 끝까지 읽을 수 있습니다.
> 회문(palindrome) 문제에서 출발해, 오토마타가 PS 전반에서 어떻게 쓰이는지까지 넓혀 봅니다.

---

## 0. 한 줄 요약

> **오토마타는 "문제를 유한개의 상태(state)와 전이(transition)로 바꿔서, 그 위에 DP·그래프탐색·행렬거듭제곱을 얹는 모델링 도구"이다.**

중요한 점부터 박아두겠습니다.

- 그리디·DP·그래프탐색은 **"푸는 전략(paradigm)"** 입니다.
- 오토마타는 그 전략들과 **같은 층이 아닙니다.** 오토마타는 **"문제를 상태기계로 환원하는 모델링"** 이고, 실제 계산은 대개 DP나 그래프탐색이 합니다.

그래서 이 보고서의 목표는 두 가지입니다.

1. "상태기계로 본다"는 게 무슨 뜻인지 몸으로 느끼기
2. 대표적인 **오토마타 기법 알고리즘**들의 지도를 그리기

---

## 1. 오토마타란 무엇인가 — 자판기로 이해하기

오토마타(유한 상태 기계, FSM)는 딱 세 가지로 이루어집니다.

1. **상태(state)** : 지금 내가 어떤 상황인가 (유한개)
2. **전이(transition)** : 입력 한 글자가 들어오면 어느 상태로 가는가
3. **시작/수락 상태** : 어디서 시작하고, 어떤 상태로 끝나면 "성공"인가

자판기를 떠올려 보세요.

```
[0원] --100원--> [100원] --100원--> [200원] --100원--> [300원=음료 나옴]
```

- 상태 = 지금까지 넣은 금액
- 입력 = 동전
- 전이 = 동전 넣으면 금액 상태가 바뀜

여기서 핵심 통찰:

> **"지금까지의 과정 전부를 기억할 필요 없이, 현재 상태 하나만 알면 충분하다."**

100원을 한 번에 3개 넣었든, 한 개씩 세 번 넣었든 **상태 `[300원]`은 똑같습니다.** 과거를 유한개의 상태로 **압축**하는 것 — 이게 오토마타의 전부이자, DP와 만나는 지점입니다.

---

## 2. 첫 번째 깨달음: "회문은 유한 오토마타로 못 잡는다"

회문(palindrome) 문제에서 출발했으니 짚고 갑시다. 흔한 오해:

> "회문도 상태기계로 인식하면 되지 않나?"

**안 됩니다.** `A...A`, `AB...BA`, `ABC...CBA` 처럼 회문은 **앞부분을 전부 기억**했다가 뒤에서 거꾸로 맞춰봐야 합니다. 길이가 무한히 길어질 수 있으니, **유한개의 상태로는 기억이 부족**합니다.

- 회문 언어는 **정규(regular)가 아니라 문맥자유(context-free)** 입니다.
- 유한 오토마타(DFA)로는 불가능하고, **스택을 가진 오토마타(푸시다운 오토마타)** 가 있어야 합니다.

그럼 회문 문제에 쓰는 "**Eertree(회문 트리)**"는 뭐냐? 그건 "임의 회문 인식기"가 아니라, **특정 문자열에 등장하는 회문들을 노드로 색인한 자동자**입니다. 개념이 다릅니다(8장에서 다룸).

---

## 3. 오토마타 기법의 본진 ① — 문자열 자동자

문자열에서 "지금까지 패턴을 어디까지 맞췄나"를 상태로 두면, 매칭이 곧 상태 전이가 됩니다.

### 3.1 KMP = 패턴 1개짜리 DFA

KMP의 **실패함수(failure function)** 가 사실은 작은 DFA입니다.

- 상태 = "패턴의 앞에서 몇 글자까지 맞췄나" (`0 ~ m`)
- 입력 한 글자가 들어오면:
  - 맞으면 → 다음 상태로 +1
  - 틀리면 → **실패링크를 타고 후퇴**해서 다시 시도
- 상태 `m`에 도달 = 패턴 발견

```cpp
// 실패함수 = "맞추다 틀렸을 때 어디로 후퇴하나"
vector<int> BuildFail(const string& p) {
    int m = p.size();
    vector<int> fail(m, 0);
    for (int i = 1, j = 0; i < m; ++i) {
        while (j > 0 && p[i] != p[j]) j = fail[j - 1]; // 후퇴(전이)
        if (p[i] == p[j]) fail[i] = ++j;
    }
    return fail;
}
```

`while`로 후퇴하는 부분이 바로 "전이가 없으면 더 짧은 상태로 점프"하는 자동자 동작입니다. **이 후퇴 아이디어가 아래 모든 문자열 자동자의 뿌리**입니다.

### 3.2 Aho–Corasick = 패턴 여러 개짜리 DFA

패턴이 여러 개일 때 KMP를 한 번에 처리합니다.

- 모든 패턴을 **트라이(trie)** 로 합친다 = 상태들
- KMP의 실패링크를 트라이에 일반화한 **failure link** 를 건다
- 텍스트를 한 번 훑으며 상태를 전이 → 등장하는 모든 패턴을 한 방에 검출

"금지 단어 목록"이 등장하는 문제의 표준 도구입니다.

### 3.3 Suffix Automaton (SAM) = 부분문자열 인식 최소 DFA

한 문자열 S의 **모든 부분문자열**을 받아들이는 최소 크기 DFA입니다. `O(n)`에 만들어지고,

- 서로 다른 부분문자열 개수 세기
- 최장 공통 부분문자열(LCS)
- 부분문자열 등장 횟수

같은 문제를 우아하게 풉니다.

### 3.4 한눈에

| 자동자 | 인식 대상 | 한 줄 정체 |
|---|---|---|
| KMP | 패턴 1개 | 패턴 DFA + 실패링크 |
| Aho–Corasick | 패턴 다수 | 트라이 + 실패링크 |
| Suffix Automaton | 모든 부분문자열 | 최소 DFA |
| Eertree | 모든 회문 | 트리 + 접미사링크(8장) |
| Bitap(Shift-And) | 패턴(오차 허용) | NFA를 비트마스크로 시뮬레이션 |

## 4. 오토마타 기법의 본진 ② — 자동자 + 다른 패러다임

실전에서 제일 자주 나오는 형태입니다. **자동자를 깔아두고 그 위에서 익숙한 알고리즘을 돌립니다.**

### 4.1 자동자 + DP (가장 흔함)

#### (a) 자리수 DP (Digit DP)

"`0 ~ N` 중 조건을 만족하는 수의 개수"를 셀 때, **수를 한 자리씩 읽는 작은 상태기계** 위에서 DP를 합니다.

- 상태 = (현재 자리, N에 딱 붙어있나(tight), 조건 관련 추가 상태…)
- 전이 = 이 자리에 숫자 `d`를 놓는다

```cpp
// 예시: 0~N 중 '연속된 같은 숫자'가 없는 수의 개수
string N;                 // 상한을 문자열로
int dp[20][11][2];        // [자리][직전숫자][tight]
int Go(int pos, int prev, int tight) {
    if (pos == (int)N.size()) return 1;
    int& ret = dp[pos][prev][tight];
    if (ret != -1) return ret;
    ret = 0;
    int hi = tight ? (N[pos] - '0') : 9;     // tight면 N의 자리까지만
    for (int d = 0; d <= hi; ++d) {
        if (d == prev) continue;             // 조건: 직전과 같으면 스킵
        ret += Go(pos + 1, d, tight && (d == hi));
    }
    return ret;
}
```

"`tight`"라는 상태가 핵심인데, 이게 바로 **"지금까지 N과 똑같이 따라왔는가"를 유한 상태로 압축**한 것입니다. 전형적인 오토마타+DP.

#### (b) Aho–Corasick + DP

"금지 단어들을 **포함하지 않는** 길이 L 문자열 개수" →
AC 자동자의 각 노드를 상태로 두고 `dp[길이][노드]` 로 전이. 금지어 끝 노드는 빼고 셉니다.

```
dp[i+1][next(node, c)] += dp[i][node]   // 단, next가 위험노드면 제외
```

### 4.2 자동자 + 그래프탐색

DFA를 **그래프**로 보면 노드=상태, 간선=전이입니다. 그러면:

- **BFS** → "이 패턴을 부분문자열로 갖는 **최단** 문자열", "특정 상태 도달 가능한가"
- Aho–Corasick에서 "위험 노드(금지어 끝)"를 피해 가는 경로 탐색

### 4.3 자동자 + 행렬 거듭제곱

전이를 인접행렬 `A`로 만들면 **길이 n짜리 경로의 수 = `Aⁿ`** 입니다.

- "패턴을 피하는 **길이 10⁹** 짜리 문자열 개수" 처럼 길이가 천문학적일 때
- 상태 수가 `k`면 `O(k³ log n)` 으로 해결

> DP의 점화식이 "상태 → 상태 선형결합" 이면, 그건 행렬 한 번으로 표현되고, 거듭제곱으로 가속할 수 있다는 신호입니다.

## 5. 오토마타의 고향 — 컴파일러/이론

PS 밖에서도 오토마타는 어디에나 있습니다.

- **렉서(어휘 분석)**: 정규표현식 → NFA(**Thompson 구성**) → DFA(**부분집합 구성, subset construction**) → 최소화(**Hopcroft**). 모든 정규식 엔진·컴파일러의 토대.
- **파서(구문 분석)**: 문맥자유 문법 → **푸시다운 오토마타**. LR 파싱이 곧 "상태 스택을 가진 자동자"입니다.

회문이 푸시다운 오토마타를 필요로 한다는 2장의 이야기가 여기서 이어집니다.

## 6. "오토마타로 모델링하라"는 신호 — 냄새 맡는 법

다음 냄새가 나면 오토마타를 의심하세요.

1. **과거 전체가 유한개의 상태로 요약된다.** ("지금까지 패턴을 k글자 맞췄다" / "N과 여기까지 똑같이 왔다")
2. **입력 한 단위마다 상태가 정해진 규칙으로 전이된다.**
3. **금지/허용 패턴**, **부분문자열 포함 여부**, **자릿수 조건** 같은 제약이 등장한다.
4. 길이가 **엄청 크다(10⁹+)** → 전이행렬 거듭제곱 신호.

모델링이 끝나면, 실제로 푸는 건 늘 하던 **DP / BFS / 행렬거듭제곱**입니다.

## 7. 전체 지도

```
풀이 전략(같은 층):   그리디 · DP · 그래프탐색 · 분할정복 · ...
                       ↑ 오토마타는 이 층이 아니라 ↓
모델링 도구:           오토마타 = 문제를 (상태 + 전이)로 환원
                          │
        ┌─────────────────┼──────────────────────┐
   문자열 자동자          자동자 + DP            자동자 + 그래프/행렬
   (KMP, Aho-Corasick,   (Digit DP,            (DFA 위 BFS 최단경로,
    SAM, Eertree,         Aho-Corasick + DP,    전이행렬 Aⁿ 카운팅)
    Bitap)                DFA 문자열 카운팅)
```

| 기법 | 결합 패러다임 | 대표 문제 |
|---|---|---|
| KMP | (자체) | 단일 패턴 매칭, 실패함수 응용 |
| Aho–Corasick | + DP/BFS | 다중 패턴, 금지어 회피 카운팅 |
| Suffix Automaton | (자체) | 서로 다른 부분문자열 수, LCS |
| Eertree | (자체) | 서로 다른 회문 수·빈도 |
| Digit DP | + DP | 조건 만족 수 세기 |
| 전이행렬 거듭제곱 | + 선형대수 | 초장문 문자열 카운팅 |

## 8. 부록 — Eertree(회문 트리) 요약

회문 문제에서 만나는 "회문 자동자"입니다. 한 문자열의 **서로 다른 회문 부분문자열 전부**를 `O(n)`에 색인합니다.

- 노드 = 서로 다른 회문 하나. 저장값: `len`, 전이 `next[c]`(=`cPc`), 접미사링크(가장 긴 진회문 접미사)
- **가상 루트 2개**가 트릭:
  - 홀수 루트 `len = -1` → c 붙이면 길이 1 회문 생성
  - 짝수 루트 `len = 0` → c 붙이면 길이 2 회문 생성
  - `len = -1`이라는 "불가능한 길이"로 홀/짝을 분기 없이 통합 처리
- 삽입: `last`(현재 접두사의 최장 회문 접미사)에서 접미사링크를 타고 후퇴(**KMP와 같은 amortization으로 전체 `O(n)`**)
- 얻는 것: 서로 다른 회문 개수(노드 수−2), 최장 회문 길이, 각 회문 등장 횟수

> **언제 쓰나**: 단순히 "최장 회문 길이"만 필요하면 **Manacher `O(n)`** 나 **중심 확장**이 더 간단합니다. Eertree의 진가는 **"회문 집합 자체(개수·빈도)"** 를 다뤄야 할 때 나옵니다.

---

## 9. 마무리

- 오토마타는 답을 내는 전략이 아니라 **문제를 상태기계로 보는 눈**입니다.
- "과거를 유한 상태로 압축할 수 있는가?"를 물어보세요. 그렇다면 상태·전이를 정의하고, 그 위에 **DP·BFS·행렬거듭제곱**을 얹으면 됩니다.
- 문자열이면 KMP→Aho-Corasick→SAM→Eertree, 카운팅이면 Digit DP/전이행렬 — 이 지도만 머리에 있으면 충분합니다.
